# Bounded C-Finite Polynomial Map Benchmarks

These benchmarks are generated by conjugating a polynomially solvable map
`h` with a polynomial automorphism `phi`, giving
`p = phi^{-1} o h o phi`.

Each source map `h` has only eigenvalue `1`, so the closed forms are polynomial
rather than exponential. This keeps the assertions focused on the bounded
C-finite polynomial-map reasoning instead of on SMT support for exponential
terms.

The files `bcf_01`, `bcf_06`, `bcf_07`, `bcf_09`, and `bcf_10` are
lower-degree invariant-based conjugates intended as the main comparison set for
the current prototype. The files `bcf_02`--`bcf_05` and `bcf_08` use larger
state spaces or more aggressive nonlinear polynomial automorphisms and are
useful as stress cases.

Each `.yml` file marks `unreach-call.prp` as `true`.

For the main comparison set, each benchmark also has three loop-bound variants:

- `_b20`: `n <= 20`
- `_b1000`: `n <= 1000`
- `_bmax`: `n <= 2147483647`

The `_bmax` variants prevent the result from being explained by concretely
running a small bounded loop.

Regenerate these variants with:

```bash
python3 ArithExe/benchmarks/bounded-cfinite/generate_bound_variants.py
```

Create assertion-level copies, where each output program contains exactly one
post-loop assertion, with:

```bash
python3 ArithExe/benchmarks/split_bounded_cfinite_assertions.py
```

The assertion-level output is written to
`ArithExe/benchmarks/bounded-cfinite-single-assert`.

Run the main comparison subset with:

```bash
python3 ArithExe/benchmarks/run_loop_lit_benchmark.py \
  --benchmark-dir ArithExe/benchmarks/bounded-cfinite \
  --metadata-dir ArithExe/benchmarks/bounded-cfinite \
  --match 'bcf_01_*_b*' --match 'bcf_06_*_b*' --match 'bcf_07_*_b*' \
  --match 'bcf_09_*_b*' --match 'bcf_10_*_b*' \
  --bounded-cfinite enabled \
  --output ArithExe/benchmarks/results/bounded-cfinite-main-enabled
```

Change `--bounded-cfinite enabled` to `--bounded-cfinite disabled` for the
baseline run without the bounded C-finite polynomial-map method.
